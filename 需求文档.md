# 产品需求文档 (PRD)：Flutter 本地与 WebDAV 视频播放器 (Android)

## 1. 需求概述

**核心目标**：开发一款基于 Flutter 的 Android 端高级视频播放器应用，旨在为用户提供统一的本地与网络（WebDAV）视频管理和播放体验。

**使用场景**：用户使用该应用扫描和管理手机本地存储的视频（如相册录像、下载的电影），同时连接家庭 NAS 或云盘（通过 WebDAV 协议）直接串流播放网络视频。

**核心亮点**：基于 MediaStore 的极速本地扫描、支持高阶超分着色器（Anime4K）、原生级别的并发缩略图生成方案，以及安全的 WebDAV 统一管理。

## 2. 功能需求 (Functional Requirements)

### 2.1 导航与全局框架 (P0)

- **F1.1 底部导航栏**：包含四个 Tab：首页（本地媒体）、网络共享（WebDAV）、收藏、设置。
- **F1.2 状态保持**：切换 Tab 时保持各页面的滚动状态和层级状态。

### 2.2 本地媒体库模块 (P0)

- **F2.1 媒体扫描 (P0)** ：通过 Android 原生 `MediaStore` API 扫描本地视频。
- **F2.2 相册聚合 (P0)** ：按“相册/桶（Bucket）”对视频进行分组聚合展示。
- **F2.3 媒体元数据落库 (P0)** ：将扫描到的相册信息、视频元数据（时长、分辨率、路径、大小等）存入本地 Isar 数据库，支持快速检索。
- **F2.4 收藏功能 (P1)** ：支持将本地视频与 WebDAV 网络视频标记为“收藏”，并在“收藏”Tab 中统一集中展示。

### 2.3 缩略图处理引擎 (P1)

- **F3.1 原生缩略图提取 (P1)** ：通过原生代码调用 FFmpeg 获取视频关键帧作为封面/缩略图。
- **F3.2 并发任务队列 (P1)** ：在 Flutter 侧实现缩略图提取的并发任务队列控制，避免阻塞 UI 或引发 OOM。
- **F3.3 缓存策略 (P1)** ：建立稳定的图片缓存路径策略（如区分本地/网络视频 Hash），支持清理缓存。

### 2.4 核心视频播放器 (MediaKit/mpv) (P0)

- **F4.1 基础播放与UI (P0)** ：全屏沉浸式播放，提供包含播放/暂停、进度条、时间显示的控制面板。
- **F4.2 手势交互 (P1)** ：屏幕左侧上下滑动调节亮度，右侧上下滑动调节音量，左右滑动控制进度快进/快退。
- **F4.3 播放控制 (P1)** ：支持倍速播放调节、画面比例切换（适应屏幕/拉伸/原始等）。
- **F4.4 高级图像处理 (P2)** ：集成超分着色器（如 Anime4K GLSL），提供画质增强开关。 **（GLSL 文件随 App 安装包内置打包）**
- **F4.5 截图保存 (P2)** ：一键截取当前视频帧并保存至系统相册。
- **F4.6 记忆播放 (P1)** ：记录视频上次播放的进度，下次打开时无缝续播。

### 2.5 WebDAV 网络模块 (P0)

- **F5.1 账户管理 (P0)** ：支持 WebDAV 账号的增、删、改、查，允许多个服务器配置。
- **F5.2 目录浏览 (P0)** ：支持浏览 WebDAV 目录，提供面包屑导航，支持设置“根路径限制”。
- **F5.3 文件检索与排序 (P1)** ：支持当前目录下的文件名搜索，支持按名称、时间、大小等维度排序。
- **F5.4 媒体筛选与播放列表 (P1)** ：自动过滤非视频文件，点击视频后自动将当前目录下的其他视频构建为播放列表（支持自动连播）。

### 2.6 系统设置模块 (P1)

- **F6.1 UI 主题设置 (P1)** ：支持亮色/暗色模式切换，及跟随系统，状态持久化。
- **F6.2 播放器默认设置持久化 (P1)** ：

  - 默认倍速
  - 默认画面比例
  - 手势快进逻辑（如滑动单位距离对应的秒数）
  - 记忆进度开关
  - 自动播放下一个开关

## 3. 非功能需求 (Non-Functional Requirements)

- **NFR1 性能 (Performance)** ：

  - MediaStore 本地扫描万级视频文件耗时不超过 3 秒。
  - 缩略图并发生成需限制最大线程数（建议依据 CPU 核心数），滑动列表时需保障 60fps 流畅度，避免卡顿。
- **NFR2 安全 (Security)** ：

  - WebDAV 的凭据（账号、密码）**必须**使用安全存储介质（如 Android KeyStore / flutter\_secure\_storage）加密保存，禁止明文落库。
- **NFR3 可用性 (Usability)** ：

  - 播放器支持防误触锁屏功能。
  - 网络不佳时，WebDAV 播放需有明确的缓冲 Loading 提示。

## 4. 边界与约束 (Boundaries & Constraints)

- **平台限制**：当前仅针对 Android 平台设计与优化，原生插件（MediaStore、FFmpeg、存储权限）暂不考虑 iOS 兼容性。
- **格式限制**：播放能力及超分着色器强依赖于 `MediaKit/mpv` 的支持范围。若部分冷门格式 mpv 无法解码，应用端不作额外软解开发。
- **不做的功能**：一期暂不涉及局域网 SMB/DLNA 扫描、不涉及在线字幕自动下载功能、不涉及视频剪辑与格式转换。

## 5. 验收标准 (Acceptance Criteria - GWT 格式)

### 场景 1：本地媒体库扫描

- **Given (前提)** ：用户首次打开 App 且授予了所有文件/相册权限。
- **When (当)** ：用户进入“首页”Tab。
- **Then (则)** ：系统在后台调用 MediaStore，UI 呈现加载态；扫描完成后，按视频所在的物理文件夹（相册）以网格视图展示，每个相册显示最新的视频缩略图和视频总数。

### 场景 2：WebDAV 视频播放与列表构建

- **Given (前提)** ：用户已成功添加并进入一个 WebDAV 目录，该目录下有 A.mp4, B.mp4, C.txt。
- **When (当)** ：用户点击 A.mp4。
- **Then (则)** ：全屏播放器启动开始缓冲 A.mp4；播放器内部自动将 B.mp4 加入待播列表（忽略 C.txt）；当 A.mp4 播放结束（且开启了自动播放），自动开始播放 B.mp4。

### 场景 3：缩略图队列与缓存

- **Given (前提)** ：相册内有 100 个新视频未生成缩略图。
- **When (当)** ：用户进入该相册并快速向下滚动。
- **Then (则)** ：屏幕可视范围内的视频优先触发 FFmpeg 缩略图生成，滑出可视区的任务被降级或取消；已生成的缩略图存入本地缓存并在下次打开时瞬间加载。

### 场景 4：Anime4K 着色器启用

- **Given (前提)** ：用户正在全屏播放一个 720p 视频。
- **When (当)** ：用户在播放控制面板点击“画质增强(Anime4K)”开关。
- **Then (则)** ：MediaKit 引擎动态加载内置的 GLSL 着色器，画面边缘变得更加锐利清晰，且播放不中断。

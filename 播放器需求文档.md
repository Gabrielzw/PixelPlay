# 详细需求文档 (PRD)：核心播放器页面

## 1. 页面概述

**页面定位**：全屏沉浸式的核心视频播放界面，承载本地与 WebDAV 网络视频的播放任务。

**核心引擎**：基于 `media_kit` (libmpv)。

**进入路径**：从“首页（本地）”、“网络共享（WebDAV）”列表点击视频，或从“收藏”、“播放历史”记录点击进入。

**页面方向**：默认根据视频画面比例自动旋转（横屏/竖屏），支持强制锁定方向。

## 2. 界面布局与视觉图层 (UI Layers)

播放器界面采用多图层叠加设计，由底层到顶层分为以下三个主层级：

### 2.1 底层：视频渲染层 (Video Render Layer)

- **功能**：纯粹的视频画面输出容器。
- **画面比例控制**：支持通过快捷键/菜单在“适应屏幕 (Fit)”、“填充拉伸 (Fill)”、“原始比例 (Original)”和“裁剪铺满 (Crop)”之间切换。

### 2.2 中层：手势与提示层 (Gesture & HUD Layer)

- **不可见手势区**：

  - **左侧 1/3 区域**：上下滑动调节**屏幕亮度**。
  - **右侧 1/3 区域**：上下滑动调节**设备音量**。
  - **全局横向区域**：左右滑动调节**播放进度**（快进/快退）。
- **HUD 提示 UI (Heads Up Display)** ：

  - 滑动触发手势时，屏幕中央或侧边需出现半透明的 UI 提示框（如：亮度百分比图标、音量百分比图标、快进的时间跨度 `[+15s / 01:20]`）。手势结束后 1 秒自动消失。

### 2.3 顶层：控制面板层 (Control Panel Layer)

- **交互逻辑**：单击屏幕空白处呼出/隐藏，无操作 3 秒后自动隐藏。
- **顶部控制栏 (Top Bar)** ：

  - [左侧] 返回按钮、当前视频标题（单行跑马灯或截断）。
  - [右侧] 截图按钮、超分增强(Anime4K)快捷开关、更多设置按钮(⋮)。
- **中央控制区 (Center)** ：

  - [居中] 播放/暂停大按钮。
  - [左侧边缘] 防误触锁屏开关（锁定后，除解锁按钮外，屏蔽一切手势和 UI 呼出）。
- **底部控制栏 (Bottom Bar)** ：

  - [上一首] / [下一首] 按钮（若当前目录有其他视频）。
  - [进度条] 拖动条 (Slider)，左侧为当前时间，右侧为总时间。
  - [倍速按钮] 点击切换或弹出倍速选择菜单（0.5x, 0.75x, 1.0x, 1.25x, 1.5x, 2.0x, 3.0x）。
  - [画面比例按钮] 快速切换画面填充模式。

## 3. 核心功能与业务逻辑

### 3.1 播放控制与列表调度

- **自动连播机制**：

  - 视频加载时，自动获取该视频同级目录下的其他视频文件，按默认排序构建为临时“播放列表”。
  - 当前视频播放至 100% 且开启了“自动播放下一个”设置时，无缝加载列表中的下一个视频。
- **记忆播放逻辑**：

  - 初始化播放时，查询数据库。若存在播放历史且进度 \> 5% 且 \< 95%，自动跳转至该进度，并短暂提示“已为您跳转至上次播放位置”。

### 3.2 高级图像处理 (Anime4K)

- **触发机制**：用户点击顶部控制栏的“Anime4K”图标。
- **效果反馈**：图标变为高亮激活态。底层调用 `media_kit` 的 `setProperty('glsl-shaders', ...)` 挂载内置的 GLSL 脚本。
- **性能降级**：若开启后检测到设备严重掉帧或引发解码错误，需自动关闭该着色器并 Toast 提示“当前设备性能不足或视频格式不兼容，已关闭画质增强”。

### 3.3 截图保存功能

- **执行逻辑**：用户点击“截图”，静默获取当前视频帧数据，保存至系统公共相册目录（如 `Pictures/YourAppName`）。
- **反馈机制**：截图成功后，在屏幕边缘（如左下角）弹出一个缩略图缩放消失的动效，或显示 Toast “截图已保存至相册”。

## 4. 状态与异常处理

### 4.1 缓冲与加载状态 (Buffering)

- **触发场景**：主要针对 WebDAV 网络视频播放。
- **UI 表现**：当 `buffer < 当前播放进度` 导致卡顿时，屏幕中央必须显示转圈 Loading 动画（覆盖在播放/暂停按钮之上）。可显示实时网速（可选）。

### 4.2 错误与异常兜底 (Error Handling)

- **解码失败/不支持的格式**：

  - **现象**：有声音无画面，或直接播放报错。
  - **处理**：展示错误蒙层，提示“视频解码失败，格式可能不受支持”，并提供“退出”按钮。
- **网络中断 (WebDAV)** ：

  - **现象**：缓冲超时（如 \> 15秒无数据）。
  - **处理**：暂停播放，提示“网络连接异常，请检查网络设置或 WebDAV 服务器状态”，并提供“重试”按钮。

## 5. 数据持久化约束 (Persistence)

在播放器页面发生以下改变时，必须实时写入本地存储（Isar / SharedPreferences）：

1. **倍速设置**：更改后，保存为全局默认倍速（视设置模块开关而定）。
2. **播放进度**：每隔 5 秒定时将当前播放进度（`position`）更新至 Isar 数据库的 `PlaybackRecord` 表。页面退出（Dispose）时执行最后一次写入。
3. **Anime4K 状态**：若用户开启，是否全局持久化取决于“设置模块”中的配置，默认仅当前视频生效。

## 6. 验收标准 (Acceptance Criteria)

- **GWT-1: 手势快进交互**

  - **Given**: 视频正在正常播放。
  - **When**: 用户在屏幕中央水平向右滑动 3 厘米。
  - **Then**: 屏幕中央显示 `[+15秒]` 的提示，进度条滑块跟随移动，松手后视频瞬间从快进后的位置继续播放。
- **GWT-2: 防误触锁屏**

  - **Given**: 唤出控制面板。
  - **When**: 用户点击左侧“锁屏”按钮。
  - **Then**: 控制面板立即隐藏，屏幕上只保留一个半透明的“解锁”按钮。此时任何单指滑动（亮度/音量/进度）、单击操作均无效，仅点击“解锁”按钮可恢复常态。
- **GWT-3: 播放进度记忆**

  - **Given**: 用户观看视频 A 到 `12:30` 时直接退出 App (Kill Process)。
  - **When**: 用户重新打开 App 并点击播放视频 A。
  - **Then**: 播放器加载完成后，进度条直接定位在 `12:30` 开始播放，并闪现提示“已恢复播放进度”。

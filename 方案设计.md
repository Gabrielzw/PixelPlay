# 方案设计

# Flutter 本地与 WebDAV 视频播放器 - 技术方案设计文档

## 1. 技术方案概述

本项目是一个以 Android 为核心平台的 Flutter 高级视频播放器应用。系统采用 **Clean Architecture（整洁架构）**  原则进行分层，核心业务逻辑（本地扫描、网络串流、媒体库管理）与 UI 渲染解耦。

关键性能瓶颈（万级本地视频扫描、并发缩略图生成、超分渲染）均下沉至 Android 原生层或 C/C++ 库（如 FFmpeg、libmpv）处理，Flutter 层主要负责跨端 UI 构建、状态管理及任务调度分发。

## 2. 技术选型表

|**领域**|**选型**||**选型理由**|
| --| --| --| -----------------------------------------------------------------------------------------------------------------------------|
|**应用框架**|**Flutter 3.x**||跨平台潜力，UI 表现力强，开发效率高，且`media_kit`支持完善。|
|**状态管理与路由**|**GetX**||语法极其精简，自带高性能依赖注入与路由管理。其响应式状态 (`Rx`) 开发效率极高，能有效减少样板代码。|
|**本地数据库**|**Isar Database**||专为 Flutter 设计的 NoSQL 库，C 语言底层，毫秒级检索万级数据，支持全文搜索。|
|**原生通信**|**Pigeon**||生成强类型接口，避免 MethodChannel 传递海量数据（如万级相册）时的拆装箱性能损耗与类型错误。|
|**视频播放器**|**media_kit (libmpv)**||唯一完美支持自定义 GLSL 着色器（Anime4K）的 Flutter 播放器，解码性能最强。|
|**缩略图生成**|**ffmpeg_kit_flutter_new**||FFmpeg抽帧速度更快，支持格式更广。使用社区维护的 new 版本能更好地适配最新的 Flutter 和 Android 构建环境，避免旧版编译报错。|
|**安全存储**|**flutter_secure_storage**||满足 NFR2，底层基于 Android KeyStore 加密 WebDAV 凭据。|
|**WebDAV**|**webdav_client**||纯 Dart 实现，轻量稳定，满足目录浏览与流媒体获取需求。|

## 3. 架构设计说明

应用采用典型的**三层架构**设计：

1. ​**表现层 (Presentation Layer)** :

   - 负责 UI 渲染和用户交互。基于 GetX 的 `Obx`​ 或 `GetBuilder` 监听响应式状态。
   - 包含：四大 Tab 页面、自定义全屏播放器 UI、设置面板。
2. ​**业务逻辑层 (Domain Layer)** :

   - 负责核心业务规则和状态管理，通过 `GetxController` 进行业务逻辑封装。
   - 包含：媒体聚合引擎（按相册分组）、播放列表管理器（自动连播逻辑）、缩略图任务调度队列（可见性优先级判定）。
3. ​**数据层 (Data Layer)** :

   - 负责数据的获取和持久化，向上层暴露 Repository 接口，通过 GetX 的 `Bindings` 注入。
   - 包含：`MediaStoreDataSource`​（调用 Pigeon 接口）、`WebDavDataSource`​、`IsarDB`​、`SecureStorage`。

​**架构图简述**：

```
[ UI 层 ] ---> (四大 Tab, 播放器面板, 缩略图列表)
   |
[ 状态层 ] ---> (GetX Controllers: 媒体库 Controller, 播放状态 Controller, 下载队列 Controller)
   |
[ 业务层 ] ---> (缩略图并发调度器, MediaKit 控制器包装, WebDAV 目录解析器)
   |
[ 数据层 ] ---> [ Isar DB ] | [ Secure Storage ] | [ WebDAV Client ] | [ Pigeon/原生 API ]
   |                                                                        |
[ 底层支撑 ] -> ( 本地存储 ) | ( Android KeyStore ) | ( NAS/云盘 )     | ( MediaStore, libmpv, FFmpeg )

```

class LocalVideo {

## 7. 关键设计决策

### 7.1 为什么万级数据扫描能控制在 3 秒内？

​**决策**​：放弃使用现成的 `photo_manager`​ 插件，手写 Android 原生 `ContentResolver` 查询逻辑。

​**实现细节**：

1. 只投影（Projection）必要的列（\_ID, DATA, DISPLAY\_NAME, BUCKET\_ID, BUCKET\_DISPLAY\_NAME, DURATION, SIZE）。
2. 使用 Pigeon 替代 MethodChannel 传输数据，跳过 JSON 序列化/反序列化，直接在底层进行二进制内存拷贝到 Dart 对象，极大减少 GC 停顿。
3. Dart 侧拿到 List 后，使用 `Isar.writeTxnSync` 进行批量同步落库（Isar 的 C 底层插入 1 万条数据只需几十毫秒）。

### 7.2 如何保证滑动 60fps 不卡顿且不 OOM (ThumbnailEngine 设计)

​**决策**​：​**按需生成 + 并发限制 + 优先级插队**。

1. ​**并发池**​：Dart 侧维护一个最大并发数为 `Platform.numberOfProcessors - 1` 的队列。
2. ​**生命周期绑定**​：在 UI 侧使用类似 `ListView.builder`​ + `VisibilityDetector`​。当 Item 进入屏幕，以高优先级（Priority\=1）入队；当 Item 滑出屏幕，如果任务未开始则直接移除，如果正在执行则不中断但丢弃 UI 渲染回调。
3. ​**缓存策略**​：生成后的缩略图以 `MD5(视频路径).jpg` 存入应用的 cache 目录。UI 直接加载本地文件系统图片，速度极快。

### 7.3 Anime4K 超分着色器集成方案

​**决策**​：通过 `media_kit` 的 mpv 属性直通加载。

​**实现细节**：

1. 将 Anime4K 的 `.glsl`​ 文件打包在 Flutter 的 `assets/shaders/` 目录下。
2. 运行时首次启用时，将 assets 中的 glsl 文件复制到 Android 沙盒目录下（因为 mpv C++ 引擎无法直接读取 apk 内的 assets）。
3. 动态配置播放器：`player.setProperty('glsl-shaders', '绝对路径/Anime4K_Restore_CNN_M.glsl')`。

### 7.4 WebDAV 密码安全管理

​**决策**：分离明文配置与敏感信息。

1. Isar DB 仅保存服务器的 URL、用户名和别名，分配一个主键 ID（如 `webdav_1`）。
2. 使用 `flutter_secure_storage`​ 调用 Android KeyStore，以 `webdav_1_pwd` 为 Key 加密存储密码。
3. 每次发起 WebDAV 连接前，先从 Secure Storage 异步解密拿到密码，再初始化 Client。

## 8. 实施计划 (Roadmap)

- **Phase 1: 基础设施与本地库 (Week 1-2)**

  - 搭建脚手架，集成 GetX 路由与状态管理、Isar 数据库。
  - 编写 Pigeon 接口与 Android MediaStore 原生代码。
  - 完成首页本地相册与视频列表 UI，使用 `GetxController` 管理加载态。
- **Phase 2: 核心播放器与图像引擎 (Week 3-4)**

  - 集成 `media_kit`，开发全屏 UI 与手势控制（音量/亮度/进度）。
  - 实现基于 FFmpeg 的并发缩略图引擎。
  - 集成并验证 Anime4K GLSL 着色器。
- **Phase 3: WebDAV 与扩展功能 (Week 5)**

  - 实现 WebDAV 账号管理（安全存储）与目录文件浏览。
  - 完善后台自动连播待播列表构建逻辑。
  - 实现进度记忆、本地/网络视频收藏功能。
- **Phase 4: 性能调优与测试 (Week 6)**

  - 针对 1 万+ 本地视频进行压测及 Isar 查询优化。
  - 快速滑动列表时的内存泄漏排查（Profile 模式）。
